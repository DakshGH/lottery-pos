# Licensing & anti-theft — how to actually protect this

## The honest truth first
A web app (or any app) **runs on the customer's machine**, so its code can be read and modified. **No client-side check is unbreakable.** Anyone who says otherwise is selling snake oil. What we *can* do is make stealing it harder than paying for it, and detect/cut off abusers. The single most effective protection is to **keep the brains on a server you control.**

Three models, from most to least secure:

### 1. Hosted SaaS — *recommended, the only truly strong option*
You host the app **and** an account/login + subscription server. Data and auth live on **your** server. A store literally cannot use the app without an active, paid login, because every meaningful action goes through your API.
- ✅ Best anti-piracy, best recurring revenue, you control updates, you see all usage.
- ⚠️ Needs a backend (Node/DB) and the app becomes online-first (less offline).
- This is how LottoReco, PDI, etc. operate.

### 2. License key + 15-minute heartbeat — *implemented here (client half)*
The app still runs client-side but **phones home every 15 minutes** to your license server, sends usage, and **locks itself** if the subscription is invalid/expired/revoked. A 7-day offline grace window keeps stores running through internet outages.
- ✅ Stops casual copying and non-payment; gives you the usage data + remote kill-switch you asked for.
- ⚠️ A technical user could edit the JS to skip the check. Good enough to start; pair with #1 later.
- This is what `src/core/license.js` does today. It is **OFF until you set a server URL + key** (so trials run free).

### 3. Packaged desktop app (Electron/Tauri) + signed license + heartbeat
Harder to tamper than raw web JS, can bind to hardware, code can be minified/obfuscated. Still not unbreakable, but raises the bar. We already have an Electron wrapper in `desktop/`.

## The heartbeat contract (server you need to build)

The client (`license.js`) sends every 15 min:

```http
POST https://your-server/validate
Content-Type: application/json

{
  "app": "lottery-pos",
  "key": "LP-7K2M-9QXA",
  "deviceId": "dev_ab12cd34",        // unique per install
  "ts": "2026-06-19T14:03:00Z",
  "tz": "America/New_York",
  "usage": { "bins": 6, "packs": 142, "days": 31, "storeName": "Maple Street Mart" }
}
```

Your server replies:

```json
{ "valid": true, "plan": "yearly", "validUntil": "2027-06-19T00:00:00Z" }
```

- `valid:false` (or a past `validUntil`) → the app **locks** on the next check.
- Record every call → you get a live list of who's using the app, which device, last seen, store name (exactly the "data of them using our application" you wanted).
- To cut someone off: mark their key revoked/expired in your DB → their next heartbeat locks them. That's the **remote kill-switch**.

### Minimal server example (deployable free on Cloudflare Workers / Render / Fly)

```js
// POST /validate  — sketch; back it with a real DB (Postgres/KV) in production
const keys = {
  'LP-7K2M-9QXA': { plan: 'yearly', validUntil: '2027-06-19', revoked: false, devices: [] },
};
export async function handler(req) {
  const { key, deviceId, usage } = await req.json();
  const rec = keys[key];
  await logHeartbeat(key, deviceId, usage, new Date()); // store for your admin dashboard
  if (!rec || rec.revoked) return json({ valid: false });
  // bind key to a max number of devices (anti-sharing)
  if (!rec.devices.includes(deviceId)) {
    if (rec.devices.length >= 2) return json({ valid: false, reason: 'device_limit' });
    rec.devices.push(deviceId);
  }
  const valid = new Date(rec.validUntil) > new Date();
  return json({ valid, plan: rec.plan, validUntil: rec.validUntil });
}
```

### Anti-sharing measures the server should enforce
- **Device binding**: cap each key to N devices (`deviceId`). 3 stores sharing one key → 4th device is rejected.
- **Expiry**: `validUntil` from their plan (6-month / yearly). Auto-locks when the term ends unless renewed.
- **Revocation**: a `revoked` flag = instant remote kill-switch.
- **Anomaly flags**: many devices, impossible travel (timezone jumps), usage from regions you didn't sell to → review/revoke.
- **Key format**: short, human-typable (`LP-XXXX-XXXX`), generated by you, never guessable.

## How to turn enforcement ON
In the app: **Settings → License & subscription** → paste the **License server URL** + the customer's **License key** → Activate. (Or it's entered on the lock screen.) Leaving the URL blank = trial mode, no enforcement — which is what your friend's test link uses now.
